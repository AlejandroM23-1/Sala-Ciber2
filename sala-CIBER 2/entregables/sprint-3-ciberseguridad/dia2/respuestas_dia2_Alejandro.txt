¿El hardening del Día 1 modifica un archivo permanente?
Sí, porque al editar el docker-compose.yml estamos haciendo un cambio en un archivo de configuración que reside en el disco. No es algo volátil; Docker lee ese
archivo cada vez que levanta el contenedor.

¿El cambio de 127.0.0.1 debería persistir?
Debería persistir siempre. Como el archivo está guardado con esa IP local, el servicio siempre se levantará en modo privado a menos que alguien vuelva a editar 
el archivo manualmente.

¿Cuál es la diferencia entre un puerto abierto y un puerto expuesto?
Un puerto abierto es el que tiene una aplicación escuchando, pero un puerto expuesto es el que el firewall permite ver desde fuera. Puedo tener el puerto de
OpenClaw abierto solo para mí en 127.0.0.1 y así no estará expuesto a nadie más en la red.

¿Por qué importa saber qué otros servicios están corriendo?
Porque cada servicio abierto es una entrada potencial. Si otro programa del sistema es vulnerable, un atacante podría entrar por ahí y luego intentar 
saltar hacia mis datos de OpenClaw o mis tokens de API.


Diferencia de riesgo:
Al no usar VM y correrlo directamente en mi Windows, el riesgo es que el agente tiene acceso más directo a mi hardware y archivos personales. Si el servicio es 
hackeado, el atacante ya está dentro de mi host principal.

Red Pública vs Privada:
En modo Privado, Windows intenta ser "amigable" y se deja ver por otros dispositivos del WiFi. En modo Público, Windows se vuelve invisible y bloquea las conexiones 
entrantes, lo cual es obligatorio aquí en el campus para que nadie detecte mi puerto de OpenClaw.

¿En qué archivos podrían haber escrito su API key?
Principalmente en el docker-compose.yml, en algún config.json o incluso en archivos de texto de notas si los guardé en la misma carpeta.

¿Han hecho algún commit que pueda contener la API key? ¿Cómo lo verificarían?
Espero que no, pero para estar seguro uso git log -p. Este comando me deja ver el contenido histórico de cada cambio. Si la clave se filtró a GitHub, cualquier bot 
la robaría en segundos y gastaría mi crédito de OpenRouter, por eso es un riesgo financiero crítico.

¿Qué es un .env?
Es un archivo local que guarda mis claves secretas para que la aplicación las use sin que el código fuente las tenga escritas. Es una forma de separar la
configuración de la programación.

¿Qué pasa si agregas .env al .gitignore después de commitearlo?
El archivo se queda en GitHub. El .gitignore solo funciona para archivos que Git nunca ha visto. Si ya se subió una vez, Git lo seguirá rastreando hasta que 
limpie el caché de la base de datos de Git manualmente.

¿Por qué el firewall y el binding son capas distintas?
El binding es una decisión de la aplicación (OpenClaw) para decidir a quién escucha (solo a mí en 127.0.0.1). El firewall es una decisión del sistema operativo 
(Windows) para decidir si deja que alguien de la red siquiera toque mis puertos. Son dos capas de defensa: si una falla, la otra todavía me protege.
